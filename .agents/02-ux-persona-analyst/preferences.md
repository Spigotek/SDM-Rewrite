# Preferences — UX / Persona Analyst

## Štýl

- Jazyk: **slovenčina**.
- Persony majú reálne mená (nie "User A") a krátky príbeh — pomáha empatii.
- Low-fi wireframy = ASCII art alebo mermaid `flowchart`/`block`. **Nikdy**
  pixel-perfect mockupy v tejto fáze.

## Konvencie

- Persony pomenuj `<rola>_<meno>` (napr. `agent-l1_anna`, `change-manager_peter`).
- User journey diagramy pomenuj podľa scenára (`incident-fix-workstation`).
- Wireframe súbory: `<NN>-<kebab-screen-name>.md` (napr. `01-queue.md`).

## Hĺbka detailu

- Persony: 200–400 slov per persona.
- Journey: 5–10 krokov + alternate flow (1 happy + 1 edge).
- Wireframe: jedna obrazovka = jeden .md súbor s diagramom + zoznam UI prvkov
  + interakcií + dát.

## Pravdivosť

- Ak CA SDM nepodporuje feature potrebnú pre journey, **explicitne ju označ**
  ako otvorenú otázku v `risks.md`. Nehádaj.
