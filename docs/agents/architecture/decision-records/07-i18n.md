## Kontext

GOAL §5 NFR a §11 vstup: **SK + EN** ako production-ready jazyky. Žiadne
ďalšie jazyky v MVP.

Komplikácie:
- **Statické UI stringy** sú riešiteľné per-app catalog (`portal.sk.json`,
  `workspace.sk.json`).
- **Dynamické hodnoty z CA SDM** (status labels, kategórie, priority labels)
  sú per-tenant konfigurovateľné v EN (alebo lokalizovateľné v CA SDM
  natívne — overiť). UX risk R-102: "i18n knižnica musí podporovať ICU
  MessageFormat + fallback na backend-provided label".
- **Persona names a placeholder data** — v UI prejdú cez Design System
  storybook (nie production).
- **Persona language**: SK persony (Lucia, Anna, ...) sú interný dev artefakt
  — bez i18n (risk R-013).
- **Locale-aware formatting**: dátumy, čísla, currency (v post-MVP), relatívne
  časy ("pred 2 hodinami"). ICU MessageFormat.
- **RTL**: nie je v scope (SK + EN sú LTR).

## Rozhodnutie

**`@sdm/i18n` package** s týmto contractom:

```ts
// @sdm/i18n/index.ts
type Locale = "sk" | "en";

interface I18n {
  locale: Locale;
  t(key: string, vars?: Record<string, unknown>): string;     // ICU MessageFormat
  formatDate(date: string | Date, opts?: Intl.DateTimeFormatOptions): string;
  formatNumber(n: number, opts?: Intl.NumberFormatOptions): string;
  formatRelative(date: string | Date): string;                // "pred 2 h"
  // Backend-provided dynamic labels — passthrough s fallback
  dynamic(value: string | { sk?: string; en?: string }): string;
}

// React binding (alebo ekv. v iných frameworkoch)
const I18nProvider: React.FC<{ locale: Locale; children: React.ReactNode }>;
const useTranslation: () => I18n;
const Trans: React.FC<{ i18nKey: string; vars?: ... }>;
```

**Implementácia**:
- Knižnica (finálne v r2): **`i18next` 23 + `react-i18next` 14 + `i18next-icu`
  + `@formatjs/intl-messageformat`**.
- `@sdm/i18n` ostáva tenký adaptér (P7) — exporty `I18n`, `I18nProvider`,
  `useTranslation`, `Trans` cez re-export z `react-i18next` + custom `dynamic()`
  helper.
- **Catalogy** sú JSON v `packages/i18n/catalogs/<app>/<locale>.json`.
  Catalog-y per app (`portal`, `workspace`) sú samostatné chunky;
  shared catalog (`shared/<locale>.json`) má spoločné kľúče (status,
  buttons, errors).
- **Loading**: bootstrap natiahne default locale; switch jazyka načíta druhý
  catalog lazy + `I18nContext` set + persistencia v localStorage prefs.
- **Dynamické hodnoty**:
  - Ak BFF posiela `string` (CA SDM raw) → `dynamic(value)` vráti raw.
  - Ak BFF posiela `{ sk: "...", en: "..." }` (post-MVP nice-to-have) →
    `dynamic(value)` vyberie podľa active locale s fallback na `en`, potom
    na `sk`, potom prvý dostupný kľúč.
- **Pluralization, gender** cez ICU MessageFormat:
  ```
  "queueCount": "{count, plural, =0 {Žiadne tickety} one {1 ticket} few {# tickety} other {# ticketov}}"
  ```

## Dôsledky

**Pozitívne**:
1. **Štandardný ICU MessageFormat** — všetky moderné i18n knižnice ho
   podporujú; ľahká migrácia ak Tech Stack rozhodne inak.
2. **Per-app catalog chunking** — portál nesťahuje workspace stringy.
3. **Dynamic value adapter** — UX risk R-102 vyriešený. CA SDM hodnoty
   passthrough alebo dispatched on shape.
4. **Date / number formatting** cez native `Intl` API — žiadny moment.js
   (deprecated).
5. **Locale stored per user** — preferences (ADR-04) persistujú locale,
   tenant switch ho neresetuje.

**Negatívne**:
1. **Catalog maintenance** — preklady SK ↔ EN sa musia udržovať. Mitigácia:
   `tools/i18n-check.ts` script v CI overí, že každý kľúč v EN má variant
   v SK a opačne.
2. **Initial bundle obsahuje 1 default locale** — extra ~50–150 kB raw
   catalog (gzipped < 30 kB). Acceptable.
3. **Dynamic value resolution** je krehká pri schema-less raw CA SDM strings
   — tím sa musí dohodnúť, ktoré polia sú translatable.

## Alternatívy

### A) `react-i18next` natívne

**Prečo nezvolené ako finále, len ako možný impl**:
- `react-i18next` je framework-bound (React). Ak Tech Stack vyberie Vue, ide
  do iného nástroja. Preto `@sdm/i18n` je adaptér nad konkrétnou knižnicou.

### B) Vlastná i18n vrstva bez ICU

**Prečo zamietnuté**:
- Bez plurals / gender / select — slovenský plural má 4 formy (`=0`,
  `one`, `few`, `other`). Bez ICU sa nezaobídeme.

### C) Backend-side i18n (BFF posiela hotové stringy)

**Prečo zamietnuté pre MVP**:
- BFF musí render-ovať podľa user locale → BFF musí mať catalog → tým
  duplikujeme catalog na server aj client.
- Statické stringy sa lepšie spravujú v FE (translator tooling, lokalizačné
  služby).
- Dynamické backend hodnoty rieši `dynamic()` adapter, ktorý je dostatočný.

### D) Žiadne i18n, len EN

**Prečo zamietnuté**:
- GOAL §5 jasne hovorí SK + EN. SK je primárny pre väčšinu užívateľov.
