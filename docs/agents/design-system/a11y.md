# A11y — WCAG 2.1 AA pre SDM-Rewrite

> **Cieľ.** WCAG 2.1 Level AA je **minimum** pre obe aplikácie (portal +
> workspace). Označené sú kritériá, kde mierime na AAA, ako aj klíčové
> per-komponent implementácie. Nie je to zoznam-checklistu na "zaškrtnúť";
> každý komponent z [`components.md`](./components.md) má svoje špecifické
> a11y poznámky.
>
> **Audience.** Frontend implementéri (06), QA agent (09), security agent (05).
> Tento dokument **nie je tutoriál** WCAG — predpokladáme znalosť princípov.

## Globálne pravidlá

### G-01 Sémantický HTML pred ARIA

ARIA je **fallback**, nie default. Pravidlo `<button>` nad `<div role="button">`.
Príklad: `IconButton` je native `<button>` s `aria-label`. Bulk action chip
je `<button aria-pressed>`, nie `<div>`.

### G-02 Klávesnicová prístupnosť

Každá akcia dosiahnuteľná myšou MUSÍ byť dosiahnuteľná z klávesnice.
Workspace má dokumentované klávesové skratky v `?` overlay (povinné per
WCAG 2.1.1 + 3.3.5).

**Predpoklad** — v `?` overlay zobrazujeme:

- App-wide shortcuts (Cmd+K, T tenant switcher, ? overlay sám).
- Per-view shortcuts (Queue: j/k/Enter/t/r/c/e/a/Esc).
- Editor shortcuts (KB Editor: Cmd+B, Cmd+I, Cmd+K link).

### G-03 Focus management

- **Visible focus** — vždy. Custom focus ring (`shadow.focus.brand`)
  alebo native browser ring. `outline: 0` bez náhrady = automatický fail.
- **Logical tab order** — DOM order = visual order. Žiadne
  `tabindex > 0`.
- **Skip link** — "Preskočiť na hlavný obsah" je prvý focusable element
  na page. Visually hidden until focused.
- **Modal focus trap** — focus ostáva v modali; on close vráti sa na trigger.
- **Roving tabindex** — `DataTable` rows, `Tabs`, `Calendar` cells používajú
  pattern: 1 row je `tabindex=0`, ostatné `tabindex=-1`. Šípky pohyb.

### G-04 Žiadna len-farba info

WCAG 1.4.1 (Use of Color, AA). Status badges majú farbu + ikonu + text.
Priority chips majú farbu + symbol (🔴🟠🟡🟢) + label ("High"). Link
v texte je color-distinct **aj** underline.

### G-05 Kontrast

WCAG 1.4.3 (Contrast Minimum, AA): text ≥ 4.5:1, large text (≥ 18 pt / 14 pt
bold) ≥ 3:1. UI komponenty a graphics ≥ 3:1 (1.4.11 AA).

### G-06 Touch target size

WCAG 2.5.5 (AAA) — 44 × 44 px. Cieľ pre **portal mobile** (Lucia má 30 %
mobile usage). Workspace desktop má 32 px (kompromis kvôli density).

### G-07 Reduced motion

`@media (prefers-reduced-motion: reduce)` → všetky `motion.duration.*` na
`instant` (0 ms). Špecifické:

- Skeleton shimmer → static placeholder.
- Modal / drawer slide → instant fade.
- Toast slide-in → instant fade.
- GSAP animations v Agent Trace (ak by sa použili) → disabled.

### G-08 Tabidn̆ / language attribute

`<html lang="sk">` alebo `<html lang="en">` per user preference (WCAG 3.1.1).
Mixed-language content (anglický KB článok v SK profile) má `lang="en"`
na container element (WCAG 3.1.2).

### G-09 Časové limity

WCAG 2.2.1 (Timing Adjustable, A). **Žiadne hard timeouts** pre user input.
Session timeout (security) má warning 2 min pred expiration s extend možnosťou.

Auto-dismiss toasts pauzuje pri hover/focus (WCAG 2.2.4 AAA).

### G-10 Auto-update / live regions

`aria-live="polite"` pre passive updates (toast success, draft auto-saved).
`aria-live="assertive"` pre kritické (error, breaking change). Polling-driven
updates (queue refresh) sa **neoznamujú**, pretože by spamovali screen reader.
Zmena statusu otvoreného ticketu → polite announce "Status changed to Open".

### G-11 i18n SK + EN

Stringy v UI nikdy nehardcoded v komponente. Wrapper komponent renderuje
key cez i18n knižnicu (rozhodne 06). Diakritika SK (`ľĺčďť`) musí render
bez fallback font replacement — Inter ju má natívne.

### G-12 Screen reader landmarks

Každá obrazovka má min: `<header>`, `<main>`, `<nav>` (kde aplikovateľné),
`<aside>` (workspace context panel), `<footer>` (mobile portal). Skip-link
preskočí na `<main>`.

### G-13 Heading hierarchia

Jedno `<h1>` per page (page title). Headings nepreskakujú levels (h1 → h2 → h3,
nie h1 → h3). `VisuallyHidden` heading je akceptovateľný pre maintenance hierarchie.

### G-14 Form errors

Inline pod každým poľom (`role="alert"` + spojený cez `aria-describedby`).
Žiadne modal alerts pre validáciu. Submit zlyhal → scroll na first error +
focus.

### G-15 Drag-drop alternative

WCAG 2.5.7 (Pointer Gestures, A). Každý drag-drop má alternative button:

- `FileUpload` dropzone má aj "vyber zo zariadenia" button.
- Calendar drag-resize (v1+) má aj Edit modal s start/end inputs.
- KB editor image drop má aj file picker button + paste shortcut.

### G-16 Hover-only info → keyboard alternative

WCAG 1.4.13 (Content on Hover or Focus, AA). Tooltips, hover cards (requester
contact card) sú dostupné aj cez `:focus` na trigger element.

## WCAG 2.1 AA — kritérium-by-kritérium

Tabuľka pokrýva všetky AA success criteria. Niektoré nepriamo aplikujú
(napr. 1.2.x media — žiadne audio/video v MVP).

### Principle 1: Perceivable

#### 1.1 Text Alternatives

| Kritérium | Level | Ako splníme |
|---|---|---|
| 1.1.1 Non-text Content | A | Všetky obrázky majú `alt`. Dekoratívne `aria-hidden="true"`. Icons systému Lucide majú `aria-label` ak sú samostatné. Form inputs majú `<label>`. |

#### 1.2 Time-based Media

| Kritérium | Level | Ako splníme |
|---|---|---|
| 1.2.1 Audio-only / Video-only (Prerecorded) | A | MVP žiadne. v1+ KB embedded video bude mať transcript. |
| 1.2.2 Captions (Prerecorded) | A | Detto. |
| 1.2.3 Audio Description / Media Alternative | A | Detto. |
| 1.2.4 Captions (Live) | AA | N/A (žiadny live stream). |
| 1.2.5 Audio Description (Prerecorded) | AA | v1+ KB video bude mať audio description. |

#### 1.3 Adaptable

| Kritérium | Level | Ako splníme |
|---|---|---|
| 1.3.1 Info and Relationships | A | Sémantické HTML (headings, lists, `<dl>` pre CI atribúty). ARIA len kde sémantika nestačí. |
| 1.3.2 Meaningful Sequence | A | DOM order = reading order. CSS flex/grid nemení sémantický poriadok. |
| 1.3.3 Sensory Characteristics | A | Inštrukcie neodvolávajú sa len na tvar/farbu/polohu. ("Klikni červené" → "Klikni Submit"). |
| 1.3.4 Orientation | AA | Layout funguje v portrait aj landscape. Workspace odporúča desktop ale neforcuje. |
| 1.3.5 Identify Input Purpose | AA | `autocomplete` attribute na formoch (name, email, tel). |

#### 1.4 Distinguishable

| Kritérium | Level | Ako splníme |
|---|---|---|
| 1.4.1 Use of Color | A | Status/priority kombinujú farbu + ikonu + text. Linky majú underline (G-04). |
| 1.4.2 Audio Control | A | Žiadny auto-play audio. |
| 1.4.3 Contrast (Minimum) | AA | Validované per token-pair v G-05. Min 4.5:1 text, 3:1 UI/large. |
| 1.4.4 Resize Text | AA | UI funguje pri 200 % zoom bez loss of content. Tested na portal forms + workspace queue. |
| 1.4.5 Images of Text | AA | Žiadne. Všetok text je real text (vrátane buttons/badges). |
| 1.4.10 Reflow | AA | Layout reflows na 320 px viewport bez horizontal scroll (portal). Workspace requires lg viewport, ukáže banner pod 1024 px. |
| 1.4.11 Non-text Contrast | AA | UI komponenty (borders, focus ring, badge borders) ≥ 3:1. |
| 1.4.12 Text Spacing | AA | Komponenty nelámu sa pri line-height 1.5, letter-spacing 0.12em, word-spacing 0.16em, paragraph-spacing 2× line-height. |
| 1.4.13 Content on Hover or Focus | AA | Hover content (tooltips, popovers): dismissable (Esc), hoverable (pohyb na popover OK), persistent (zostane viditeľný kým user nedismisne). |

### Principle 2: Operable

#### 2.1 Keyboard Accessible

| Kritérium | Level | Ako splníme |
|---|---|---|
| 2.1.1 Keyboard | A | Všetky interaktívne elementy ovládateľné z klávesnice (G-02). |
| 2.1.2 No Keyboard Trap | A | Modal focus trap má vždy Esc / close button escape. Žiadny iný trap. |
| 2.1.4 Character Key Shortcuts | A | Single-key shortcuts (`j`, `k`, `t`, ...) sú aktívne **iba** keď focus nie je v input/textarea. Implementácia per `useHotkeys` filter na `event.target.tagName`. |

#### 2.2 Enough Time

| Kritérium | Level | Ako splníme |
|---|---|---|
| 2.2.1 Timing Adjustable | A | G-09. Žiadne hard timeouts pre user input. |
| 2.2.2 Pause, Stop, Hide | A | Žiadne animované content (kromě skeleton shimmer, ktorý `prefers-reduced-motion` disabled). Toast auto-dismiss pauzuje pri hover/focus. |

#### 2.3 Seizures

| Kritérium | Level | Ako splníme |
|---|---|---|
| 2.3.1 Three Flashes or Below Threshold | A | Žiadne flash animácie. |

#### 2.4 Navigable

| Kritérium | Level | Ako splníme |
|---|---|---|
| 2.4.1 Bypass Blocks | A | Skip-link "Preskočiť na hlavný obsah" + landmarks. |
| 2.4.2 Page Titled | A | `<title>` dynamicky update per route (např. "INC-1042 — Notebook reštart · Acme HQ · SDM Workspace"). |
| 2.4.3 Focus Order | A | DOM order = visual order. |
| 2.4.4 Link Purpose (In Context) | A | Linky majú visible text alebo `aria-label` opisujúci destination. Žiadne "click here". |
| 2.4.5 Multiple Ways | AA | Search (Cmd+K), navigation menu, breadcrumb — 3 spôsoby dosiahnutia. |
| 2.4.6 Headings and Labels | AA | Headings opisujú topic; labels opisujú purpose (G-13). |
| 2.4.7 Focus Visible | AA | Custom focus ring vždy viditeľný (G-03). |

#### 2.5 Input Modalities

| Kritérium | Level | Ako splníme |
|---|---|---|
| 2.5.1 Pointer Gestures | A | Žiadne path-based / multi-point gestures requirements. Calendar drag-resize má alternative (G-15). |
| 2.5.2 Pointer Cancellation | A | Buttons fire on `mouseup` / `click`, nie `mousedown`. Cancel cez move-off-target funguje. |
| 2.5.3 Label in Name | A | Visible label = `aria-label` accessible name. ("Save" button má `aria-label="Save"`, nie `"submit-btn"`.) |
| 2.5.4 Motion Actuation | A | Žiadne motion-triggered features (žiadny shake-to-undo). |

### Principle 3: Understandable

#### 3.1 Readable

| Kritérium | Level | Ako splníme |
|---|---|---|
| 3.1.1 Language of Page | A | `<html lang="sk">` / `<html lang="en">` (G-11). |
| 3.1.2 Language of Parts | AA | Anglické KB články v SK profile: `lang="en"` na container (G-11). |

#### 3.2 Predictable

| Kritérium | Level | Ako splníme |
|---|---|---|
| 3.2.1 On Focus | A | Žiadny context change pri focus (Select otvorí len pri Enter / Space, nie pri tab-in). |
| 3.2.2 On Input | A | Form submit len cez explicit button. Tenant switch má confirm. |
| 3.2.3 Consistent Navigation | AA | Top bar identický cez všetky obrazovky. Nav items v rovnakom poradí. |
| 3.2.4 Consistent Identification | AA | Rovnaký komponent (`Button`, `Badge`) má rovnaký label/aria napríč obrazovkami. |

#### 3.3 Input Assistance

| Kritérium | Level | Ako splníme |
|---|---|---|
| 3.3.1 Error Identification | A | Inline errors `role="alert"` + spojené cez `aria-describedby` (G-14). |
| 3.3.2 Labels or Instructions | A | Každý input má visible label. Required marker `*`. |
| 3.3.3 Error Suggestion | AA | Errors poskytujú návrh ("Email musí obsahovať @"). |
| 3.3.4 Error Prevention (Legal, Financial, Data) | AA | Submit ticket / approve change má confirm step alebo undoable action (cancel ticket within 5 min). |

### Principle 4: Robust

#### 4.1 Compatible

| Kritérium | Level | Ako splníme |
|---|---|---|
| 4.1.1 Parsing | A | Valid HTML (auto-validované linter pri build). |
| 4.1.2 Name, Role, Value | A | Native elements alebo ARIA. Custom kompozity (Combobox, Tabs) majú complete ARIA implementation. |
| 4.1.3 Status Messages | AA | `aria-live` regions pre status (G-10). |

## High-contrast theme (HC)

Návrh: tretia téma popri light/dark. Detail v [`theming.md`](./theming.md).

- Žiadne shadows.
- Borders na všetkých interactive elements (button, input, badge) ≥ 2 px.
- Text contrast ≥ 7:1 (AAA target).
- Focus ring ≥ 3 px solid border.
- Severity colors preserved ale s explicit border-color tokens.

## Testing strategy hand-off (do 09)

Aspoň tieto kontroly v CI:

1. **Static** — `axe-core` cez `@axe-core/playwright` na každý route v e2e.
2. **Lighthouse a11y score** — ≥ 95 na portal home, workspace queue, ticket
   detail. CI fail < 90.
3. **Manual screen reader pass** — pred release per major feature (VoiceOver
   macOS, NVDA Windows). Sample flow: portal new ticket + portal KB read +
   workspace ticket reply.
4. **Keyboard-only walk-through** — recorded video / checklist per release
   (3.3.4 → submit, escalate, approve change cez klávesnicu).
5. **Reduced motion** — Playwright `prefers-reduced-motion` runtime, validuje
   že žiadna `motion.duration` > `instant`.
6. **Contrast** — `axe-core` má built-in. Token-pair audit (G-05) v unit testoch
   (Style Dictionary token contrast plugin).

## Otvorené závislosti

- `[06-tech-stack-selector]` Headless component library (Radix / Headless UI
  / Ark UI) musí mať out-of-the-box ARIA implementations pre Combobox, Tabs,
  Dialog. Cieľ: nemusíme implementovať ARIA ručne pre kompozity.
- `[05-security]` SSO redirect flow MUSÍ rešpektovať WCAG 2.4.3 — focus po
  návrate na app je na first interactive (alebo `<h1>`).
- `[05-security]` 2FA step-up flow pre Peter mobile approve — UI prompt musí
  byť keyboard-only operable.
- `[08-devex-devops]` Pre-commit hook pre `axe-core` na changed routes.
  Block PR keď a11y critical violation.
- `[09-qa-test-strategy]` Šablóna pre manual SR test sheet (VoiceOver/NVDA),
  per release.
- `[03-domain-modeller]` SK preklady status labels musia byť **rozšírené**
  aria-label kontextom — napr. "V riešení" (visible) → "Status: V riešení,
  prevzala Anna pred 12 minútami" (aria-label).
